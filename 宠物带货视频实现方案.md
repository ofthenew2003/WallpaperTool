# 宠物带货视频 — 实现方案

## Context

在现有 AI 工具集基础上新增"AI宠物带货视频"工具。用户上传宠物照片 + 产品图片，AI 自动生成 15 秒带货短视频。项目当前有壁纸生成器（单次 API 调用），视频生成是多阶段流水线，需新增类型、API 层、store、composable、5 个组件。

## 技术选型

| 层 | 方案 | 理由 |
|---|---|---|
| 场景图生成 | 复用 SiliconFlow 图片生成 API | 已有 client，只需构造包含参考图的 prompt |
| 图转视频 (I2V) | 抽象接口，默认接 Kling AI | Kling 图转视频质量好，中文市场友好；接口可替换 Runway / Minimax |
| 视频拼接 | 服务端 API 拼接（优先），降级 `ffmpeg.wasm` | 服务端拼接最可靠；`ffmpeg.wasm` ~30MB 按需加载，不增加首包 |
| 样式预设 | 5 种视频风格标签 | 可爱/炫酷/自然/搞笑/高级感，映射为 prompt 后缀 |

## 流水线（5 阶段）

```
上传验证 → 生成场景提示词 → 生成场景图 → 图转视频 → 拼接+下载
  (1)         (2)              (3)          (4)         (5)
```

- **(1) 上传验证** — 校验两张图片 base64，裁剪/缩放到标准尺寸
- **(2) 场景提示词** — 调 LLM 根据产品名+卖点生成 3 段分镜提示词（中文），每段对应一个 5s 镜头
- **(3) 场景图生成** — 用 SiliconFlow 文生图为每个分镜生成一张 1024×1024 场景图（pet+product 融合）
- **(4) 图转视频** — 每张场景图调用 I2V API 转 5s 视频片段
- **(5) 拼接下载** — 3 段 5s 拼接为 15s MP4，加背景音乐，浏览器下载

### 进度映射

| 阶段 | progress % | 文案 |
|---|---|---|
| 上传完成 | 5 | — |
| 生成场景提示词 | 15 | AI 正在构思分镜… |
| 生成场景图 | 30→60 | AI 正在绘制第 N 张场景… |
| 图转视频 | 60→90 | AI 正在生成第 N 段视频… |
| 拼接导出 | 95 | 正在合成最终视频… |
| 完成 | 100 | 生成完成！ |

### 1. 类型扩展 — `src/types/index.ts`（追加）

```typescript
export type VideoStyle = 'cute' | 'cool' | 'natural' | 'funny' | 'luxury'
export type VideoStage = 'idle' | 'uploading' | 'prompts' | 'images' | 'videos' | 'stitching' | 'done'

export interface ScenePrompt  { index: number; prompt: string }

export interface PetVideoRequest {

export interface PetVideoResult {
  id: string
  petImage: string
  productImage: string
  productName: string
  videoUrl: string
  videoBlob?: Blob
  scenes: SceneClip[]
  timestamp: number
}
```

### 2. API 层

**`src/api/video.ts`**（新建）
- `generateScenePrompts(productName, sellingPoints, style)` → 调 SiliconFlow LLM chat/completions 生成 3 组分镜提示词。若不可用，回退到本地模板拼接。
- `generateSceneImage(prompt, referencePetBase64, referenceProductBase64)` → 调 SiliconFlow `/images/generations` 文生图，在 prompt 中附带参考描述。
- `imageToVideo(imageBase64, prompt, duration)` → 调 Kling I2V API，返回视频 URL。抽象为接口，方便切换。
- `stitchVideos(clipUrls[])` → 拼接多个视频 URL 为单一视频 URL（优先服务端，降级客户端）。

**`src/api/client.ts`**（追加）
- 新增 `videoApiClient` 实例指向视频 API base URL（可配置）

### 3. Store — `src/stores/petVideo.ts`（新建，参考 `generation.ts`）

state: petImage, productImage, productName, sellingPoints, style,
       stage, progress, error, scenes[], result
actions: setXxx(), reset()

### 4. Composable — `src/composables/usePetVideo.ts`（新建，参考 `useGeneration.ts`）

```
generate(request: PetVideoRequest) → 驱动 5 阶段流水线
  1. 校验输入
  2. await generateScenePrompts()  → 更新 scenes
  3. for scene: await generateSceneImage() → 更新 scenes[i].imageUrl
  4. for scene: await imageToVideo() → 更新 scenes[i].videoUrl
  5. await stitchVideos() → 生成最终 videoUrl
  6. 存入 store.result
regenerate(sceneIndex?) → 重试失败的分镜
downloadVideo() → 触发浏览器下载
```

## 不做的

- 服务端后端（纯前端 SPA + 第三方 API）
- 视频历史记录持久化（v1 仅内存中保存当前结果，v2 可加 localStorage）
- 自定义背景音乐上传（v1 使用预设静音或单一默认音轨）
